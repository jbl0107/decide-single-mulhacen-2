Translation subsystem


Why have we chosen the translation subsystem?

Translating a Django application into any language and allowing it to be detected depending on the requests of your users is a very important feature nowadays, In addition, it is very useful to learn to do this since most of the pages that we find on the Internet give you the option of translating them. In this section we will explore a bit how to do it. To carry out this translation task we needed to have a Django project, which we have used the decide project from the repository provided in the subject of Evolution and Configuration Management. In addition, we need to have gettext installed, which is a module that reads the language used by the system and from this loads the corresponding translation file (*.mo) changing the text strings to the system language.


Steps to perform:

  1. Make sure we have translations enabled in our settings. To do this, we need to make the following changes to the settings.py: 
                                  
                              
                                   LANGUAGE_CODE = 'es'
                                   
                                   TIME_ZONE = 'UTC'
                                   
                                   USE_I18N = True
                                   USE_L10N = True
                                   USE_TZ = True
                                   
                                   LOCALE_PATHS = [
                                          os.path.join(BASE_DIR, 'locale')
                                   ]
                                   
                                   LANGUAJES = [
                                         ('en', 'English'),
                                         ('es', 'Spanish'),
                                   ]

    For this case we have defined that our project is by default in English.

  2. Once what is indicated in the previous step has been done, we should add the following line 'django.middleware.locale.LocaleMiddleware' to the settings.py of our project, it is important Place this middleware between SessionMiddleware and CommonMiddleware.

    In this case, what we have done is tell Django to use the LocaleMiddleware which will take care of choosing the appropriate language for each user based on information extracted from the request.

  3. The next step is to mark the texts that we want to translate within our project, for this there are different ways depending on the text that we are going to translate.

    Traduce in templates

    To explain how to translate templates, suppose we have an index.html in templates/some_app/index.html

    We need to adapt this file so that it can be translated:

    • We would have to add the sentence {'%trans texto_que_se_quiere_traducir %'} in each text that we want to translate.

      • In addition, in each html that we want to introduce a translation we must include the i18n templatetags with {'% load i18n %'}.

  4. Later you would have to create a folder called locale in the path some_project/locale.

    Having everything configured, now yes, we proceed to create the translation files. For this we use the following command from the project path: django-admin makemessages -l es

    This will create the translation files for the Spanish language, which is the one we have used for our project.

  5. With the commands executed in the previous step, a .po file will be generated in the locale folder.
    Translate .po files

    These files are the ones we will use to specify to Django how to translate each word in a certain language, so simply for each of the strings that appear in msgid we can specify its translation with strings msgstr

  6. Next we proceed to compile the messages created with the previously executed command.
    Compilar archivos de traducci√≥n

    Compile translation files

    Once we have our complete translation files, we will proceed to compile them with the following command: django-admin compilemessages When we compile our files, we are ready to use our project in the specified language. Every time one of our users consults our project it will appear in their language. For testing purposes, we can change the language of the project to Spanish and observe the results.

  7. How did we select the language?

    For this we have implemented a drop-down html code, which allows us to select the language to which we want to translate and, later, once selected, we must click on a button that says "Go" which translates the entire html page to the selected language

Cosas a tener en cuenta:

  • Every time we add an html in our project we must modify the file urls.py where we have to add a line like this path('path/', view_method_name) , and also import said view in our file urls.py in this way from .views import view_method_name
  • You also have to modify the views.py file and add a function for each new html file that you add, this function will contain a line like this: return render(request, 'rutaNueva /file.html')
